## 3. 翻譯規則

### 3.1 保留不動的內容

以下內容翻譯時**不可修改**：

- **顏色代碼**：`&a`、`&e`、`&f`、`&7`、`&8`、`&0`、`§x§1§c...` 等
- **佔位符**：`%player_name%`、`%template%`、`%time%`、`%count%` 等
- **PAPI 變數**：`%receiver%`、`%sender%`、`%title%` 等
- **YAML 結構**：key 名稱、縮排、列表格式
- **命令**：`/sweetmail`、`[close]`、`[player]`、`[console]` 等
- **特殊標記**：`<hover:show_text:>`、`&8&m...&r`（刪除線）
- **cancel**：取消操作的關鍵字，保持英文小寫
- **all/unread**：收件匣篩選參數，保持英文小寫

### 3.2 翻譯原則

1. **以面板介面為最高依據**：messages.yml 的用語必須與面板一致
2. **統一用語**：同一概念在所有檔案中使用相同的繁體詞彙
3. **語氣自然**：使用台灣玩家習慣的用語，避免中國用語
4. **保留原意**：不改變功能含義，只做語言轉換
5. **格式對齊**：保持與原文相同的排版和換行

### 3.3 常見錯誤

| ❌ 錯誤 | ✅ 正確 | 原因 |
|---|---|---|
| 寄件匣 | 已寄郵件 | 面板標題為「已寄郵件」 |
| 發送郵件 | 寄送郵件 | 面板按鈕為「寄送郵件」 |
| 正文 | 文本 | 面板欄位為「文本」 |
| 書與筆 | 書和羽毛筆 | 面板 lore 為「書和羽毛筆」 |
| 聊天欄 | 聊天室 | 面板 lore 為「聊天室」 |
| 實數 | 數值 | 用語統一 |
| 標為已讀 | 設為已讀 | 面板為「全部設為已讀」 |
| 打开 | 開啟 | 繁體中文標準用語 |
| 发送 | 寄送 | 面板用語 |

## 4. 介面設計規範

### 4.1 面板標題格式

所有面板標題使用統一格式：

```
§0§l蜂聊郵政&f » §x§1§1§c§1§c§1§c§l[功能名稱]
```

- `§0§l` = 黑色粗體（主標題）
- `&f »` = 白色箭頭分隔符
- `§x§1§c§1§c§1§c§l` = 灰色粗體（副標題）

### 4.2 物品名稱格式

功能按鈕使用統一格式：

```
§x§9§d§a§f§f§f> §f[名稱]
```

- `§x§9§d§a§f§f§f` = 淡藍色（#9dafff，即 `--sky` 色）
- `> ` = 箭頭前綴
- `§f` = 白色文字

### 4.3 Lore 格式

```
&8&m                                               &r     ← 分隔線
§7　[說明文字] : &f%value%                                  ← 內容行
&8&m                                               &r     ← 分隔線
§7*點擊 [操作說明]                                        ← 操作提示
```

- `&8&m` = 深灰色刪除線（用作分隔線）
- `§7` = 灰色（說明文字）
- `&f` = 白色（數值/變數）
- `§7*` = 灰色星號（操作提示前綴）

### 4.4 系統訊息格式

```
&#CBE0FD系統 &8| &7[訊息內容]
```

- `&#CBE0FD` = 淡藍色（系統前綴）
- `&8|` = 深灰色分隔符
- `&#FF3300❌` = 紅色叉號（錯誤）
- `&#00FB9A✔` = 綠色勾號（成功）

### 4.5 介面佈局

- 每行 **9 個圖示**，最多 **6 行**
- 黑色玻璃板（`BLACK_STAINED_GLASS_PANE`）作為空白填充
- 灰色玻璃板（`GRAY_STAINED_GLASS_PANE`）作為次要填充
- 障礙物（`BARRIER`）作為返回按鈕
- 功能按鈕使用對應的 Minecraft 物器材質

## 6. 新增翻譯流程

1. 取得原始簡體中文檔案
2. 對照本指南第 2 節術語表進行翻譯
3. 保留所有顏色代碼、佔位符、YAML 結構
4. 確認用語與面板一致（第 3.3 節常見錯誤）
5. 交付前逐項檢查第 3.1 節保留內容是否完整
