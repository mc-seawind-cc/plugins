| 色名 | 色碼 | 用途 |
|---|---|---|
| 天空藍 | `&#9dafff` / `§x§9§d§a§f§f§f` | 主色調、導航項目前綴 |
| 薰衣草紫 | `§x§b§9§f§c§b§9` | 導航項目名稱 |
| 淺綠 | `&#72FA93` | 指令/快捷鍵標籤 |

| 白色 | `&f` / `§f` | 主要文字 |
| 灰色 | `&7` | 說明、備註、Lore |

### 2.2 色碼格式

| 格式 | 範例 | 說明 |

## 3. 介面設計規範

### 3.1 面板標題格式

**統一格式：**
```
§0§l海風選單&f » &#1c1c1c&l[功能名稱]
```

**範例：**
```
§0§l海風選單&f » &#1c1c1c&l首頁
§0§l海風選單&f » &#1c1c1c&l傳送頁
§0§l海風選單&f » &#1c1c1c&l系統 &#1c1c1c&l1&f/&#1c1c1c&l3
§0§l海風選單&f » &#1c1c1c&l傳送頁&f » &#1c1c1c&l玩家傳送頁 &#1c1c1c&l2&f/&#1c1c1c&l2
```

- `§0§l` = 黑色粗體（主標題）
- `&f »` = 白色箭頭分隔符
- `&#1c1c1c&l` = 深灰粗體（副標題）
- 多層用 `&f »` 串接
- 頁碼格式：`&#1c1c1c&l1&f/&#1c1c1c&l3`

**其他插件標題：**
```
§0§l蜂聊郵政&f » §x§1§c§1§c§1§c§l[功能名稱]           ← SweetMail
<gradient:#33EEBB:#3388BB><b>AxPlayerWarps</b></gradient> &7»  ← AxPlayerWarps
```

### 3.2 物品名稱格式

**導航項目（開啟子頁面）：**
```
name: '&#9dafff|  §x§b§9§f§c§b§9[名稱]'
```
- `&#9dafff|` = 天空藍豎線前綴
- `  ` = 兩個空格
- `§x§b§9§f§c§b§9` = 薰衣草紫色（名稱本身）

**動作項目（執行指令）：**
```
name: '&#9dafff> &f傳送至重生點　&#72FA93[&f/spawn&#72FA93]'
```
- `&#9dafff>` = 天空藍箭頭前綴
- `&f` = 白色（功能名稱）
- `　` = 全形空格（分隔）
- `&#72FA93[&f/cmd&#72FA93]` = 淺綠色標籤框，內含白色指令

**已選中/當前頁面：**
```
name: '&#9dafff|  §x§b§9§f§c§b§9首頁'
lore:
- '&7*已在此頁面'
```

### 3.3 Lore 格式

**操作提示：**
```
lore:
- '&7*點擊 開啟[頁面名稱]'
- '&7*點擊 傳送至[地點名稱]'
- '&7*點擊 [動詞][目標]'
```

**當前位置標記：**
```
lore:
- '&7*已在此頁面'
```

**分隔線（SweetMail/AxPlayerWarps 面板）：**
```
&8&m                                               &r
```

**資訊行（AxPlayerWarps）：**
```
 &#33EEBB[標題]:
 &#444444❙ &f%value%
```

### 3.4 系統訊息格式

**SweetMail：**
```
&#CBE0FD系統 &8| &7[訊息]
&#FF3300❌ = 錯誤
&#00FB9A✔ = 成功
```

**AxPlayerWarps：**
```
&#BBFFDD[成功訊息]    ← 淡綠色
&#FF4444[錯誤訊息]    ← 紅色
&#33FF33[系統訊息]    ← 綠色
```

### 3.5 介面佈局

- 每行 **9 個圖示**，最多 **6 行**
- `黑色玻璃板` — 空白填充（layout.fill）
- `GRAY_STAINED_GLASS_PANE` — 次要填充
- `BARRIER` — 返回按鈕
- `AIR` — 空白佔位
- Layout 使用數字編號對應 slot（10, 12, 13...）

### 3.6 完整物品範例

```yaml
# 導航項目
社區系統:
  material: OAK_SIGN
  stack: 1
  name: '&#9dafff|  §x§b§9§f§c§b§9社區系統'
  lore:
  - '&7*點擊 開啟社區系統頁面'
  actions:
    commands:
      - '[open] menuB1_社區'

# 動作項目（含指令標籤）
傳送至重生點:
  material: COMPASS
  stack: 1
  name: '&#9dafff> &f傳送至重生點　&#72FA93[&f/spawn&#72FA93]'
  lore:
  - '&7*點擊 傳送至重生點'
  actions:
    commands:
      - '[chat] /spawn'
```

## 4. 翻譯規則

### 4.1 保留不動的內容

| 類型 | 範例 |
|---|---|
| 顏色代碼 | `&a`、`§x§1§c...`、`&#9dafff` |
| 佔位符 | `%player_name%`、`%time%`、`%count%` |
| PAPI 變數 | `%receiver%`、`%sender%`、`%name%` |
| YAML 結構 | key 名稱、縮排、列表格式 |
| 指令 | `/pw`、`/sweetmail`、`[open]`、`[chat]`、`[close]` |
| 物品材質 | `PLAYER_HEAD`、`BARRIER`、`COMPASS` |
| 面板 ID | `menuA`、`menuB1`、`menuC` |
| 取消關鍵字 | `cancel` |
| 插件名稱 | `AxPlayerWarps`、`SweetMail` |

### 4.2 翻譯原則

1. **以已上線成品為最高依據** — 用語必須與現有介面一致
2. **統一用語** — 同一概念在所有插件中使用相同詞彙
3. **語氣自然** — 台灣玩家習慣的用語
4. **保留原意** — 不改變功能含義
5. **格式對齊** — 保持與成品相同的排版

### 4.3 翻譯流程

```
1. 取得原始檔案
2. 先翻譯 lang/messages（優先）
3. 再翻譯 config 中的可見文字
4. 最後決定 GUI 面板是否需要翻譯
5. 對照第 5 節術語表，確保用語一致
6. 確認色碼、格式與第 2-3 節規範一致
7. 存入 插件名/zh_tw/ 資料夾
```
