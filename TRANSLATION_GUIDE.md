# 海風伺服器插件翻譯與介面指南

> 最後更新：2026.05.01
> 語言：繁體中文（zh_tw）
> 維護者：海風網站助手

---

## 1. 概述

本指南為海風伺服器插件翻譯的統一規範，確保所有插件的用語一致、介面風格統一。

### 已翻譯插件

| 插件 | 對應功能 | 翻譯範圍 | 狀態 |
|---|---|---|---|
| SweetMail | 蜂聊郵政 | messages.yml + 5 個面板 | ✅ 完成 |
| AxPlayerWarps | 公共傳送點 | lang.yml（面板不翻譯） | ✅ 完成 |

### 翻譯優先序

1. **lang/messages** — 系統訊息、錯誤提示、幫助文字（玩家直接看到）
2. **config** — 分類名稱、排序名稱、預設文字（影響介面顯示）
3. **GUI 面板** — 互動介面的文字（視插件需求決定是否翻譯）

## 2. 翻譯規則

### 2.1 保留不動的內容

以下內容翻譯時**不可修改**：

| 類型 | 範例 | 說明 |
|---|---|---|
| 顏色代碼 | `&a`、`&e`、`§x§1§c...` | Minecraft 顏色格式 |
| 佔位符 | `%player_name%`、`%time%`、`%count%` | 插件變數 |
| PAPI 變數 | `%receiver%`、`%sender%`、`%name%` | PlaceholderAPI |
| YAML 結構 | key 名稱、縮排、列表格式 | 檔案結構 |
| 指令 | `/pw`、`/sweetmail`、`[close]`、`[player]` | 遊戲指令 |
| 特殊標記 | `<hover:show_text:>`、`&8&m...&r` | Minecraft 書本/聊天格式 |
| 取消關鍵字 | `cancel` | 保持英文小寫 |
| 篩選參數 | `all`、`unread` | 保持英文小寫 |
| 插件名稱 | `AxPlayerWarps`、`SweetMail` | 保留英文或 gradient 格式 |
| 物品材質 | `PLAYER_HEAD`、`BARRIER`、`CHEST` | Minecraft 物品 ID |
| 命令類型 | `[console]`、`[player]`、`[close]`、`[delay]` | 面板指令格式 |
| 物品 ID | `itemsadder-`、`mythic-`、`head-base64-` | 插件物品前綴 |

### 2.2 翻譯原則

1. **以面板介面為最高依據**：lang/messages 的用語必須與已翻譯的面板一致
2. **統一用語**：同一概念在所有插件中使用相同的繁體詞彙
3. **語氣自然**：使用台灣玩家習慣的用語，避免中國大陸用語
4. **保留原意**：不改變功能含義，只做語言轉換
5. **格式對齊**：保持與原文相同的排版和換行

### 2.3 翻譯流程

```
1. 取得原始檔案（簡體中文或英文）
2. 先翻譯 lang/messages（優先）
3. 再翻譯 config 中的可見文字
4. 最後決定 GUI 面板是否需要翻譯
5. 對照第 3 節術語表，確保用語一致
6. 檢查第 2.1 節保留內容是否完整
7. 存入插件目錄的 zh_tw/ 資料夾
```

## 4. 介面設計規範

### 4.1 面板標題格式

**SweetMail（蜂聊郵政）：**
```
§0§l蜂聊郵政&f » §x§1§c§1§c§1§c§l[功能名稱]
```

**AxPlayerWarps（公共傳送點）：**
```
<gradient:#33EEBB:#3388BB><b>AxPlayerWarps</b></gradient> &7»
```

- `§0§l` = 黑色粗體（SweetMail 主標題）
- `§x§1§c§1§c§1§c§l` = 灰色粗體（SweetMail 副標題）
- `gradient` = 漸層色（AxPlayerWarps 前綴）
- `&7»` = 灰色箭頭分隔符

### 4.2 物品名稱格式

**SweetMail：**
```
§x§9§d§a§f§f§f> §f[名稱]
```
- `§x§9§d§a§f§f§f` = 淡藍色（#9dafff，即 `--sky` 色）

**AxPlayerWarps：**
```
&#BBFFDD[說明]　　　[/pw]
```
- `&#BBFFDD` = 淡綠色
- `&#33EEBB` = 薄荷綠（主要色調）

### 4.3 Lore 格式

**通用分隔線：**
```
&8&m                                               &r
```

**SweetMail Lore：**
```
§7　[說明文字] : &f%value%
§7*點擊 [操作說明]
```

**AxPlayerWarps Lore：**
```
 &#33EEBB[標題]:
 &#444444❙ &f%value%
```
- `&#444444❙` = 深灰色豎線（資訊前綴）
- `&#33EEBB` = 薄荷綠（標題色）

### 4.4 系統訊息格式

**SweetMail：**
```
&#CBE0FD系統 &8| &7[訊息內容]
&#FF3300❌ = 錯誤
&#00FB9A✔ = 成功
```

**AxPlayerWarps：**
```
&#BBFFDD[訊息]    ← 成功/一般（淡綠色）
&#FF4444[訊息]    ← 錯誤（紅色）
&#33FF33[訊息]    ← 系統通知（綠色）
```

### 4.5 介面佈局

- 每行 **9 個圖示**，最多 **6 行**
- `BLACK_STAINED_GLASS_PANE` — 空白填充
- `GRAY_STAINED_GLASS_PANE` — 次要填充
- `BARRIER` — 返回按鈕
- `AIR` — 空白佔位

## 6. 新增插件翻譯檢查清單

翻譯新插件時，依序檢查：

- [ ] 取得原始 lang/messages 檔案
- [ ] 對照第 3 節術語表翻譯
- [ ] 保留第 2.1 節所有不可修改內容
- [ ] 確認用語與已翻譯插件一致（第 3.3 節常見錯誤）
- [ ] 決定 GUI 面板是否需要翻譯
- [ ] 如需翻譯面板，確認格式與第 4 節規範一致
- [ ] 存入 `插件名/zh_tw/` 資料夾
- [ ] 如有 config 需修改，建立 `config.patch.yml` 對照檔
- [ ] 更新本指南第 5 節翻譯記錄
- [ ] 推送到插件專案倉庫
